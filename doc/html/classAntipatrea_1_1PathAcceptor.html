Detailed Description

Interface to test whether or not a path is valid at a given time-step resolution.

Author
Erion Plaku, Amarda Shehu
Remarks
  • The path is considered valid iff every configuration along the path is acceptable.
  • For practical purposes, classes implementing this interface often discretize the path at a given time-step resolution into a sequence of configurations and deem the path acceptable iff every configuration in the sequence is acceptable.
  • The end configurations (at time = 0 and time = 1) are not checked (assumed to be acceptable).
  • This class requires access to CfgManager and CfgAcceptor to determine whether or not configurations along the path are acceptable.
