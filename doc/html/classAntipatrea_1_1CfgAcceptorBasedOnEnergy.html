Detailed Description

Motion-planning component which accepts a configuration only if its energy is below a user-defined threshold.

Author
Erion Plaku, Amarda Shehu
Remarks
  • This class requires access to CfgEnergyEvaluator component in order to evaluate the energy of the configuration.
  • The user should also set the energy threshold.

virtual void Antipatrea::CfgAcceptorBasedOnEnergy::SetupFromParams ( Params &  params)
inlinevirtual

Set the parameter values of the component from the given parameters.

Author
Erion Plaku, Amarda Shehu
Remarks
  • Function first invokes CfgAcceptor::SetupFromParams(params).
  • It then sets the energy threshold (keyword Constants::KW_EnergyThreshold).
  • When used as a goal acceptor, it uses the parameter group associated with the keyword Constants::KW_GoalAcceptorBasedOnEnergy.
  • When used as general cfg acceptor, it uses the parameter group associated with the keyword Constants::KW_CfgAcceptorBasedOnEnergy
  • The parameter value can be specified in a text file as, for example,
    CfgAcceptorBasedOnEnergy { EnergyThreshold -12.34 }
    or
    GoalAcceptorBasedOnEnergy { EnergyThreshold -100.12 }

virtual bool Antipatrea::CfgAcceptorBasedOnEnergy::IsAcceptable ( Cfg &  cfg)
inlinevirtual

Returns true iff the energy of cfg is below the user-defined threshold.

Author
Erion Plaku, Amarda Shehu
Remarks
  • The function will call CfgEnergyEvaluator component to evaluate the energy of cfg
  • It will do so, however, only if the energy of cfg has not been previously evaluated (it checks whether the energy field is undefined – if so, it computes the energy; otherwise, it accesses the energy value).
