Detailed Description

Interface for evaluating the direct and reverse cost of a path.

Author
Erion Plaku, Amarda Shehu
Remarks
  • The reverse cost of a path from A to B corresponds to the cost of the reverse path, namely from B to A.
  • Often the direct and reverse costs are the same but not always. This interface allows someone to implement nonsymmetric path-cost evaluators.

Evaluate the direct and reverse cost of a path using not only its start and end configurations but also intermediate configurations along the path.

Author
Erion Plaku, Amarda Shehu
Remarks
  • In many cases, e.g., straight-line path in Euclidean space, the path cost does not depend on the intermediate configurations but only on its start and end configurations.
  • This is the default implementation.
  • There are, however, many other cases where the path cost depends on the intermediate configurations. For example, when the path cost is related to the energy values attained by the configurations along the path.
  • When implementing this function, one can use the PathGenerator pgen to generate the intermediate configurations to better evaluate the path cost.
  • A suggested time-step resolution is also provided as an argument.
