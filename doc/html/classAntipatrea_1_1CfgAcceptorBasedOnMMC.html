Detailed Description

Motion-planning component which accepts a configuration only if the difference in energy is below accepted based on a temperature and a Metropolis test.

Author
Kevin Molloy, Erion Plaku, Amarda Shehu
Remarks
  • This class requires access to CfgEnergyEvaluator component in order to evaluate the energy of the configuration.
  • The initial temperature and adjustment factor are user parameters.

Set the parameter values of the component from the given parameters.

Author
Kevin Molloy, Erion Plaku, Amarda Shehu
Remarks
  • Function first invokes CfgAcceptor::SetupFromParams(params).
  • It then sets the energy threshold (keyword Constants::KW_EnergyThreshold).
  • Used as general cfg acceptor, it uses the parameter group associated with the keyword Constants::KW_CfgAcceptorBasedOnMMC.
  • The parameter value can be specified in a text file as, for example,
    CfgAcceptorBasedOnMMC { Temperature 70.00 }

◆ IsAcceptable()

virtual bool Antipatrea::CfgAcceptorBasedOnMMC::IsAcceptable ( Cfg &  cfg)
inlinevirtual

Returns true iff the energy of cfg is lower than the CfgSource configuration OR it passes a probabalistic test based on the current temperature.

Author
Kevin Molloy, Erion Plaku, Amarda Shehu
Remarks
  • The function will call CfgEnergyEvaluator component to evaluate the energy of cfg
  • It will do so, however, only if the energy of cfg has not been previously evaluated (it checks whether the energy field is undefined – if so, it computes the energy; otherwise, it accesses the energy value).
