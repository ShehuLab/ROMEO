Detailed Description

Motion-planning component for managing configurations.

Author
Erion Plaku, Amarda Shehu
Remarks
  • This class provides functionality to create, delete, copy, read, or print configurations.
  • The user should set the configuration dimension.
  • Why design it this way? Consider the alternative where Cfg allocates itself. Then, inside a planner/component one could write Cfg *cfg = new Cfg() to create a new configuration. This is problematic however if someone extends the configuration class, e.g., class MyCfg : public Cfg and wants to use MyCfg instead of Cfg for planning purposes. By using CfgManager, such issue is avoided since one would write a class class MyCfgManager : public CfgManager, set the allocator for the planner/component to an instance of MyCfgManager, and then write inside the planner/component Cfg *cfg = GetCfgManager()->NewCfg() .

Remarks
